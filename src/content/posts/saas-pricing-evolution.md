2014: You sell software. Customers pay per user. Everyone understands this.

2024: You sell software. Customers pay per API call, per storage used, per feature accessed. Nobody understands this.

Here's how we got here.

## The Per-Seat Era

```
┌─────────────────────────────────────┐
│         Per-User Pricing            │
│  $10/user/month                     │
│  +50 users = +$500/month            │
│  Simple. Predictable.               │
└─────────────────────────────────────┘
```

This worked. It was easy to understand. But it had problems.

- Companies with 1000 employees paying for 300 seats they don't use
- No incentive to reduce user count
- Disconnected from actual value delivered

## The Tiering Era

Basic, Pro, Enterprise. Pick your bucket. This reduced complexity but created new problems.

- Feature gating became a game
- "Contact Sales" for everything meaningful
- Companies paying for tiers they don't need

## The Usage-Based Era

```
┌─────────────────────────────────────┐
│       Usage-Based Pricing           │
│  Base fee: $50/month                │
│  + $0.001 per API call              │
│  + $0.10 per GB storage            │
│  Pay for what you use               │
└─────────────────────────────────────┘
```

This aligns incentives. More usage means more value. More value means more payment.

But it's complex. Customers struggle to predict costs. Churn increases when bills spike.

## Where We're Going

The future is probably hybrid. Some base fee for infrastructure. Usage components for value delivered. AI credits for the new thing everyone needs.

It's not getting simpler. It's getting more nuanced.

The winners will be companies that help customers understand the math.
