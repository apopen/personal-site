Junior engineers write code.
Senior engineers solve problems and make sure the same fire does not happen twice.

That is the difference.
Let me explain.

## What Juniors Do

- Implement features
- Follow specifications
- Need guidance on approach
- Focus on their code

## What Seniors Do

- Understand the why
- Make independent decisions
- Consider the system, not just their part
- Lift up others

## The Key Skills

**Technical judgment** - When to use what. When to refactor. When it's good enough.

**Communication** - Translating between technical and non-technical. Writing. Speaking.

**Ownership** - Taking responsibility for outcomes, not just tasks.

**Mentorship** - Making others better. Helping them grow.

## The Hard Part

Technical skills plateau. The higher you go, the less code you write, the more you:

- Decide what to build
- Remove obstacles
- Help others succeed
- Make tradeoffs

## Getting There

- Take on hard problems
- Think beyond your code
- Help others learn
- Communicate clearly
- Own outcomes

## The Truth

Seniority is not about years.
It is about impact and how you create it through other people.
Anyone can become senior if they focus on the right things long enough.

Code is the start.
It is not the destination.
